Security architecture

Three locks between AI and your data

Every AI query passes through three independent safety layers. If any one fails, the query is blocked. Your POS data is never modified, exported, or used for model training.

Lock 1: SQL validation Only SELECT queries pass. INSERT, UPDATE, DELETE, DROP all blocked.
Lock 2: Read-only role Database user has SELECT permission only. No writes possible at any level.
Lock 3: Read-only transaction Connection enforces default_transaction_read_only. Even a rogue query cannot write.
